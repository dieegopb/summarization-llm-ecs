# Scalable AI Processing Pipeline for Text Summarization

**Building a Scalable AI Text Processing Pipeline on AWS (ECS + SQS)**

---

## Overview
This project implements a scalable, cloud-native pipeline for automatic text summarization using AWS services. It leverages Amazon ECS (Fargate), SQS, and DynamoDB to process and store summaries generated by an AI model.

## Architecture
- **SQS**: Receives and buffers incoming text messages for summarization.
- **ECS (Fargate)**: Runs the summarization worker in Docker containers, scaling as needed.
- **DynamoDB**: Stores original texts and their summaries for later retrieval and analysis.
- **ECR**: Hosts the Docker image for the summarizer service.

## Features
- Asynchronous, scalable message processing
- Modular summarization logic (easily replaceable by other models)
- Cloud-native, serverless infrastructure
- Easy integration with other AWS services

## Getting Started

### Prerequisites
- AWS account with permissions for ECS, SQS, DynamoDB, and ECR
- Docker and AWS CLI installed
- Python 3.8+

### Setup
1. Clone this repository:
   ```bash
   git clone <repo-url>
   cd <repo-folder>
   ```
2. Configure AWS credentials:
   ```bash
   aws configure
   ```
3. Build and push Docker image:
   ```bash
   docker build -t sumarizador .
   docker tag sumarizador:latest <your-account-id>.dkr.ecr.<region>.amazonaws.com/sumarizador:latest
   docker push <your-account-id>.dkr.ecr.<region>.amazonaws.com/sumarizador:latest
   ```
4. Deploy ECS service using the latest image and configure SQS/DynamoDB as needed.

### Usage
- Envie textos para a fila SQS configurada.
- O serviço ECS irá processar e salvar os resumos no DynamoDB.
- Veja logs e status pelo CloudWatch ou Datadog.
